# add-my-skill

把用户给出的提示内容整理成本插件的一个技能。Claude Code 里也可通过 `/add-my-skill <内容>` 触发；Codex 里用自然语言触发（如「把这段整理成技能」）。

## 何时使用

- 用户想把一段说明、流程或经验沉淀成可复用的技能。
- 用户提供了要转成技能的原始内容（口语化、零散也可以）。

## 步骤

1. 从提示内容提炼一个 kebab-case 技能名 `<name>`。若 `skills/<name>/` 已存在，改用更精确的名字，不要覆盖已有技能。
2. 创建 `skills/<name>/SKILL.md`，frontmatter 必须包含：
   - `name`：与目录名一致。
   - `description`：一句话写清「这个技能做什么 + 何时触发」，末尾附触发词，便于被正确调用。
3. 正文把提示内容整理成可执行的技能说明（建议含「何时使用」「步骤」小节）；把零散、口语化的内容归纳成清晰、可复制运行的指令。
4. 大段参考资料、示例或脚本放进子目录（`references/`、`examples/`、`scripts/`），正文按需引用，保持 `SKILL.md` 精简。
5. 完成后向用户报告新建技能的相对路径和 `description`。
